The benchmark in `docs/slides.typ` already holds the two fixed references we need:
the **original-code qsyn** column and the **Vandaele** column. So a full 2×2 rerun
is wasteful — re-running Vandaele or original-qsyn reproduces known numbers. The
ablation only needs the **one new variable**: the **ported qsyn**, on each input
type. That is a **1×2** sweep — ported-qsyn × {pyzx, raw} — whose output becomes
**one added column** in the slides next to the existing `qsyn` column.

**Do not edit the existing slide numbers.** The original-qsyn and Vandaele columns
are the frozen baseline the improvement is claimed against; only *add* a column.

Steps:

1. **Run the 1×2 sweep (maintainer, on request).** `bash bench2x2/rerun-qsyn.sh` — qsyn-only,
   ported binary, same pipeline/caps as the original

   Writes to a **separate** dir `bench2x2/results-new/` (baseline
