## Phase 1 — Analysis (REDO against fastTodd.cpp)

The prior report analyzed the wrong baseline (the `qsyn::experimental` base
TODD on `dev/todd-fix`). Rewrite its Phase-1 section against `fastTodd.cpp`.
Findings already gathered (in worklog):

- **Two TOHPE paths**: `tohpe_once` (slow/reference: builds L, Z, S matrices,
  L-nullspace, `calculate_score` over S-matrices, first move) vs
  `tohpe_once_iteration` (fast: maintained **kernel** + `augmented_matrix` +
  `pivots`, incremental column add/remove via `clear_column`, scored z). This
  is Vandaele's incremental-GE TOHPE (Alg 2).
- **FastTODD move**: `fast_todd_iteration` builds the Thm-5/6 system
  `build_l`/`build_x`/`build_v`, stacks `[L | X | v]`, takes nullspace,
  enumerates `2^basis` combos (`iterate_nullspace_combinations`), rank-1
  updates, keeps first term-count-reducing move. `fasttodd_once` = TOHPE
  pre-pass to fixpoint, then one FastTODD move.
- Strategy drivers `Tohpe…::optimize` / `FastTodd…::optimize` mirror base
  TODD: `properize`, `MultiLinearPolynomial` signature snapshot, loop to
  fixpoint, Clifford correction.

Deliverable: updated Phase-1 section in the `.typ`, mapping each function to
Heyfron/Vandaele equations, plus a "current state / concerns" subsection.

## Phase 2 — Correctness & bug fixes

Pre-req: clean rebuild on `fix/fasttodd-final` (in progress; `make clean` done
to avoid stale-cache pollution from the Jun-11 `dev/todd-fix` build).

1. **Sanity + equivalence baseline.** Run `tohpe` and `fasttodd` on small
   circuits (`tof_3`, `barenco_tof_3`, `gf2^4_mult`) via a dofile:
   `qcir read … → convert qcir tableau → tableau optimize phasepoly fasttodd →
   convert tableau qcir → qcir print --stat`. Verify the optimized circuit is
   equivalent to the input (qsyn equivalence/miter), and T-count is
   ≤ base `todd`.
2. **Fix the reliability bugs** (the brief's "reliability falls short"), via
   systematic-debugging + TDD (failing test first):
   - Slow `tohpe_once` vs `tohpe_once_iteration` **divergence** — the
     `QSYN_TOHPE_TRACE*` scaffolding exists precisely to chase this. Decide one
     correct path; make them agree or drop the slow one.
   - `iterate_nullspace_combinations` is **`2^num_basis`** — exponential blow-up
     risk on large nullspaces; bound or replace with basis-only scan.
   - `FastTodd…::optimize` lacks the `has_same_signature_tensor` guard that
     `Tohpe…::optimize` has — add the correctness safety net (or justify).
   - First-move vs Vandaele **argmax** (Eq 85) move selection — confirm intended.
   - Remove leftover debug scaffolding (`QSYN_TOHPE_TRACE*`, trace helpers)
     once divergence is resolved.
3. **Regression tests.** Add `tests/tableau/dof/*.dof` + `ref/*.log` golden
   tests for `phasepoly tohpe` and `phasepoly fasttodd` on the small circuits,
   pinning T-count and equivalence. Run via `make test` (`scripts/RUN_TESTS`).

Critical files: `src/tableau/optimize/fastTodd.cpp`,
`src/tableau/optimize/todd.{cpp,hpp}`, `src/tableau/tableau_optimization.cpp`,
`tests/tableau/{dof,ref}`.
